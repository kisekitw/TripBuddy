# TripBuddy 需求差距分析 (Gap Analysis)

> 生成日期：2026-04-05
> 分析對象：TripBuddy MVP codebase vs. 需求文件

---

## 已實作 ✅

- 拖拉排序行程景點 + cascade 時間自動調整 (auto/lock 兩模式)
- 衝突偵測 (Level 0–2) + 視覺標示
- 不確定性管理：日層級變體 (Type A) + 景點層替代方案 (Type B)
- SVG-based 地圖視覺化（非互動式）
- Import modal UI (3 階段流程，mock)
- i18n (繁中 / 英文)
- 旅程清單頁 + 編輯器三欄版面

## 總結

目前 codebase 完成了約**核心 Trip Editor 的 UI 骨架**（需求 1.1–1.4 的前端展示層），但幾乎所有**後端整合、AI 功能、LINE Bot、照片管理、闖關系統**都尚未實作。

最接近完整的是衝突偵測邏輯，但解決動作仍是空殼按鈕。

| 類別 | 未實作數 | 優先度建議 |
|------|---------|-----------|
| 基礎編輯功能 (E) | 6 | 🔴 最高 — 無此功能無法真正使用 |
| 衝突解決 (C) | 5 | 🔴 高 — 衝突偵測已有，缺解決動作 |
| 匯入解析 (I) | 3 | 🟡 中 — UI 骨架已有，需接 API |
